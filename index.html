Global Tutor Academy

Global Tutor Academy is an organization providing online tuition classes for students from class 1 to 12. Our aim is to create a web application that allows students to enroll in courses and maintain their studies efficiently.

Objective

The objective of the Global Tutor Academy web application is to:

  • Facilitate Comprehensive Course and Batch Management: Provide a centralized platform for admins to manage courses, batches, and schedules, ensuring efficient organization and availability of classes.
  • Streamline Tutor and Student Interactions: Enable seamless registration and management for tutors and students, including profile updates, course assignments, and batch management.
  • Enhance Operational Efficiency: Implement intuitive dashboards for admins and tutors to monitor key metrics, handle joining requests, and access relevant notifications and updates.

User Types and Interfaces

The application will have three main types of users and four types of interfaces:

  • Users:
    • Admin
    • Tutors
    • Students
  • Interfaces:
    • Website Interface: In English & Arabic
    • Admin Interface
    • Student Interface: In English & Arabic
    • Tutor Interface

1. Admin Interface

The admin interface is designed to provide comprehensive control over the applicationâ€™s operations:

  • Dashboard: Displays key metrics, including total enrolled students, tutors, batches, and courses. It also includes tables for today's classes and joining requests, enabling quick access to important information.
  • Manage Tutors: A table listing all tutors with options to add, verify, update, or delete. Includes a search bar for filtering tutors by name, course, or phone number, and filters for verified status, course type, etc.
  • Manage Students: A table displaying all students with options to add, view more details, or delete students. Search and filter options are provided based on name, course, batch, and tutor in charge.
  • Manage Courses: Lists all courses with functionalities to add, edit, or delete courses. Searchable and filterable by course name, duration, fee, start, and end dates.
  • Manage Joining Requests: Table format showing all student joining requests with options to view, update, or delete requests. Includes search and filter capabilities by student name, course, tutor, and date range.
  • Manage Batches: Provides a table view of all batches with details and options to create, edit, or delete batches. Includes batch ID, batch name, schedule, and associated students.

2. Student Interface

The student interface focuses on providing a seamless experience for course enrollment and study management:

  • Course Browsing: Allows students to browse available courses and view details about the tutors teaching them.
  • Enrollment Process: Students can choose a tutor and request to join classes, providing necessary information like name, class, school, contact details etc.
  • Student Registration The students are supposed to create an account for themselves before joining the class. For creating an account they can use google mail registration method or create a account with a unique username and password
  • Email Notification: On successful account creation, course enrollment, the students will receive email
  • Payment:The students can pay the fee along with the joining request or when the request is accepted through their dashboard
  • Payment Method: The students can pay into the company's account through payment portals powered by Stripe/Qpay/Paypal using their debit/credit cards

Student Dashboard will contain

  • Profile Management: Students can update personal information such as username, password, email, and contact details.
  • Class Management: Displays the batch details, timetable along with meeting links, meeting password, recorded classes link, etc.
  • Course Management: Displays all information related to enrolled courses, fee payment, fee status, tutor details, batch alloted, etc.
  • Course Discontinuation Form: In the Course management section, near the enrolled courses there will be an option called "discontinue course" which on clicking a form appears with
  • Language Changer: The students can change the language of student dashboard interface to arabic or default English

3. Tutor Interface

The tutor interface provides tools for managing classes and student interactions:

  • Registration and Login: Tutors can register and log in using a secure username and password system.
  • Profile Management: Tutors can update their profiles with comprehensive details, including education, availability, and personal bio.
  • Batch Management: Allows tutors to create, edit, or delete batches. Each batch includes a unique ID, name, schedule, meeting links, and student management capabilities.
  • Class Joining Requests: Tutors manage requests from students, with options to accept, decline, or leave requests pending until further notice.
  • Student Batch Assignment: Tutors can assign students to specific batches, view batch details, and manage student participation in classes.

4. Website Interface

The website provides an intuitive interface for all users to access necessary information and functions:

  • Home Page: General overview of services and offerings.
  • About Page: Detailed information about Global Tutor Academy and its mission.
  • Course and Tutor Profiles: Detailed tutor profiles with videos, bios, and course offerings.
  • Contact Page: Easy access to contact information and a contact form for inquiries.
  • Student and Tutor Registration Pages: Step-by-step guides and links for registration and login for both students and tutors.
  • Language Changer: The viewers can change the language of the website interface to Arabic or defulat language - English
